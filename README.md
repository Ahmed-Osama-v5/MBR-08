## Overview

The **MBR-08** is an open-source Modbus RTU slave relay module intended for use in:

- Industrial I/O panels (RS-485 bus control)
- Smart building automation (lighting, HVAC switching)
- DIY IoT gateways (e.g., Raspberry Pi + Node-RED + InfluxDB + Grafana stack)
- Lab automation and test bench control

The firmware is written in bare-metal C, targeting the STM32G0B1CBTx, with a clean MCAL → HAL → Application layered architecture and zero direct HAL calls inside module logic (all HAL access is abstracted behind compile-time macros for portability).

## Firmware Architecture

### Layer Structure

```
Application (App/)
    │  Business logic: relay control
    │  Zero direct HAL calls — uses macro wrappers only
    ▼
HAL Abstraction (modbus_slave.h)
    │  modbus slave logic
    │  ...
    ▼
STM32 HAL / CMSIS (Drivers/)
    │  Generated by STM32CubeMX (.ioc file included)
    ▼
Hardware (STM32G0B1CBTx)
```

### Modbus RTU Implementation

- **Role:** Slave only
- **Interframe timing:** LPTIM1-based 3.5T detection (not software timer — accurate at all supported baud rates)
- **Supported function codes:**

  | FC   | Name                    | Direction |
  |------|-------------------------|-----------|
  | 0x01 | Read Coils              | Master → Slave (read) |
  | 0x05 | Write Single Coil       | Master → Slave (write) |
  | 0x0F | Write Multiple Coils    | Master → Slave (write) |
  | 0x03 | Read Holding Registers  | Master → Slave (read) |
  | 0x06 | Write Single Register   | Master → Slave (write) |
  | 0x10 | Write Multiple Registers   | Master → Slave (write) |

- **CRC:** Hardware-assisted or software CRC-16/IBM (Modbus polynomial `0xA001`)
- **RS-485 DE/RE:** GPIO-controlled direction pin, managed in UART TX complete callback

### EEPROM Driver

A three-file I2C EEPROM driver targets the **24LC08** (8 Kbit, 16-byte pages):

| File               | Role                                      |
|--------------------|-------------------------------------------|
| `i2c_eeprom_cfg.h` | Device address, page size, timeout config |
| `i2c_eeprom.h`     | Public API declarations                   |
| `i2c_eeprom.c`     | Implementation: read, write (page-aware), ACK poll |

Features:
- **Page-boundary-aware writes** — automatically splits writes that cross a 16-byte page boundary
- **ACK polling** after write cycles (no fixed delay)
- Follows the same Hungarian notation and module naming conventions used throughout the codebase

Persisted configuration includes: node address, baud rate index, relay power-on state.

## Communication Parameters

### Default Parameters

| Parameter    | Default       |
|--------------|---------------|
| Baud rate    | 9600 bps      |
| Data bits    | 8             |
| Parity       | None          |
| Stop bits    | 1             |
| Node address | 1             |

### Supported Baud Rates

| Index | Baud Rate  |
|-------|------------|
| 0     | 1200       |
| 1     | 2400       |
| 2     | 4800       |
| 3     | 9600       |
| 4     | 19200      |
| 5     | 38400      |
| 6     | 57600      |
| 7     | 115200     |

Baud rate is selected via GPIO input pins on boot (hardware DIP switches) and stored in EEPROM. Register-based runtime change is also supported (takes effect after reset).

## Build & Flash

### Prerequisites

| Tool                  | Version tested |
|-----------------------|----------------|
| STM32CubeIDE          | 2.1.1          |
| STM32CubeMX           | 6.11+ (`.ioc` included) |
| arm-none-eabi-gcc     | 12.3+          |
| ST-Link V2 / V3       | Any            |

### Building with STM32CubeIDE

```bash
# Clone the repository
git clone https://github.com/Ahmed-Osama-v5/MBR-08.git
cd MBR-08

# Open in STM32CubeIDE: File → Open Projects from File System
# Select the repo root directory
# Build: Project → Build All  (or Ctrl+B)
```

The `.cproject` and `.project` Eclipse CDT files are included. The build produces `MBR-08.elf` and `MBR-08.bin` under `Debug/` or `Release/`.


# Flashing via ST-Link CLI:
STM32_Programmer_CLI -c port=SWD -w Debug/MBR-08.bin 0x08000000 -rst
```

## Configuration

### Baud Rate Selection

Baud rate is read from dedicated GPIO input pins on startup. Pull the pins LOW/HIGH according to the baud rate index table above. The value is latched once during `Modbus_Init()` and written to EEPROM if changed.

## IoT Integration Example

The MBR-08 has been validated against a **Raspberry Pi 5** acting as Modbus RTU master and IoT gateway:

```
[MBR-08] ←── RS-485 ──→ [USB-RS485 adapter] ──→ [RPi 5]
                                                      │
                                              Node-RED (Modbus TCP/RTU)
                                                      │
                                              InfluxDB 2.x
                                              (org: HexaMix, bucket: relay_states)
                                                      │
                                              Grafana Dashboard
```

Node-RED `node-red-contrib-modbus` nodes can poll coil states and write relay outputs directly. InfluxDB line protocol logging provides historical state tracking per channel.
